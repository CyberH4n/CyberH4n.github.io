进程调度介绍

Created2023-03-21|Updated2023-03-21|CS操作系统
|Post View:

首先对操作系统运行的进程做出假设：

  1. 每个工作运行相同时间
  2. 所有工作同时到达
  3. 一旦开始，就保持运行直到完成
  4. 所有工作只用CPU
  5. 每个工作的运行时间已知

定义周转时间:

T周转时间=T完成时间−T到达时间T_{周转时间}=T_{完成时间}-T_{到达时间} T周转时间​=T完成时间​−T到达时间​

根据上述假设，到达时间为0

定义响应时间:

T响应时间=T首次运行−T到达时间T_{响应时间}=T_{首次运行}-T_{到达时间} T响应时间​=T首次运行​−T到达时间​

先进先出 (FIFO)

​ 也叫先到先服务(FCFS)，设ABC依次几乎同时到达，每个工作运行10s，则平均周转时间为(10 + 20 + 30) / 3 = 20。若A此时运行100s，BC运行10s，则平均周转时间为(100 + 110 + 120) / 3 = 110。

最短任务优先 (SJF)

​ 先运行最短的任务，然后运行次短，如此下去。此时上面的例子可以得到更好的平均周转时间: (10 + 20 + 120) / 3 = 50。但由于前文假设了第3条，因此如果在100s的A已经在运行的时候，10s的BC才到达，那么平均周转时间仍然很糟糕。

最短完成时间优先 (STCF)

​ SJF非抢占，通过向SJF添加抢占，我们得到最短完成时间优先或抢占式最短作业优先调度程序。每当新工作进入系统时，它就会确定剩余工作和新工作中谁的剩余时间最少，然后调度该工作。设100s的A在运行10s后，BC依次到达，它们每个运行时间都是10s，通过将BC抢占进来，对这三个工作的平均周转时间为(120 + 10 + 20) / 3 = 50。在这种情况下，A响应时间为0，B为0，C为10。但是如果3个工作几乎同时到达，那么对于稍微后到达的工作可能就需要排队，响应时间不会很好。

轮转 (RR)

​ RR在一个时间片内运行一个工作，然后切换到运行队列的下一个任务，反复执行，直到所有任务完成。RR(时间切片)必须为时钟中断周期的倍数。例如，ABC是3个同时到达的5s任务，SJF的响应时间为(0 + 5 + 10) / 3 = 5，而如果我们使用一个1s的时间片，则响应时间为(0 + 1 + 2) / 3 = 1。时间片越短，RR响应时间越好，但是如果时间片太短，上下文切换的成本会影响整体性能。考虑周转时间，RR带来的结果可能非常糟糕。

结合I/O

​ 在此放宽假设4，即所有程序都执行I/O。在A执行I/O时，抢占运行B，实现重叠，否则在I/O时阻塞状态CPU没有被利用到。
